# Тестовое задание для n8n-инженера

Привет! Это задание — проверочный кейс перед тем, как ты получишь доступ к боевому проекту Goodini. Мы хотим понять, как ты работаешь с чужим кодом и как мыслишь при рефакторинге.

## Что за проект

Goodini — это AI-пайплайны генерации видеоконтента для недвижимости. Стек: n8n, Postgres, MinIO, Redis, ffmpeg, внешние API (Replicate, Google AI Studio, ElevenLabs, OpenRouter). Всё развёрнуто на сервере в `docker-compose`.


## Структура задания

Задание состоит из двух частей плюс setup.

### Часть 1: Code Review

В папке [`part-1-code-review/`](part-1-code-review/) лежат три n8n-воркфлоу. В каждом мы заложили несколько проблем — от security до n8n-специфичных граблей.

Что нужно:
1. Импортировать каждый воркфлоу в локальный или серверный self-hosted n8n.
2. Найти все проблемы.
3. Расставить их по приоритету (`high / medium / low`).
4. Реализовать фиксы для топ-3. Любые 3 из всех найденных.

Подробности и формат сдачи — в [`part-1-code-review/README.md`](part-1-code-review/README.md).

### Часть 2: Forensic Combine Worker

В папке [`part-2-forensic/`](part-2-forensic/) находится один из наших боевых воркфлоу — `Combine Worker - MVP2.0`. Это финальная нода пайплайна: она собирает видеоклипы, накладывает озвучку и музыку, опционально оверлеит аватар и загружает результат в MinIO.

Что нужно:
1. Разобрать воркфлоу: что делает каждая группа нод, в каком порядке, какие зависимости между нодами.
2. Найти 3 потенциальных слабых места — что угодно: недостающая обработка ошибки, edge case, ресурсная утечка, плохая читаемость.
3. Предложить 2 рефакторинга с обоснованием.
4. Реализовать одну задачу из трёх на выбор — см. подробности в [`part-2-forensic/README.md`](part-2-forensic/README.md). Все три задачи связаны с ffmpeg.

### Setup

Минимальный `docker-compose` для запуска `n8n + Postgres + MinIO` локально находится в [`setup/`](setup/). Инструкция — там же.

## Что мы ждём от тебя

- Импортируешь воркфлоу, прокликиваешь, разбираешься.
- Сдаёшь два markdown-отчёта по шаблонам из каждой части: `deliverable-template.md`.
- Прикладываешь обновлённые JSON-файлы:
  - с фиксами для Part 1;
  - с реализованным рефакторингом / задачей для Part 2.
- Можно сдать одним zip-архивом, можно GitHub-репозиторием.
- Добавь `@username` в Telegram — можно файлом в репозитории.

## Чего мы не ждём

- Идеального кода. Мы смотрим на ход мысли, а не на стиль.
- Переписывания архитектуры. Если у тебя есть что сказать в духе `вообще тут надо всё переделать` — напиши это отдельным абзацем в отчёте, но фиксить не нужно.
