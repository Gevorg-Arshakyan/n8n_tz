# Часть 2: Forensic Combine Worker

В этой папке - наш реальный воркфлоу `combine-worker-mvp2.json`. Это финал пайплайна генерации видео в проекте.

Что он делает в общих чертах:
- Раз в минуту cron смотрит таблицу `jobs` на запись со статусом `queued` и `progress >= 75` (это значит, что озвучка, музыка и аватар уже готовы).
- Атомарно забирает себе одну джобу.
- Скачивает видеоклипы из MinIO.
- Через ffmpeg по SSH в контейнер n8n собирает их в один ролик.
- Опционально миксует музыку, накладывает голос, оверлеит аватар chromakey.
- Заливает финальный mp4 в MinIO.
- Обновляет `jobs.status = 'done'`, шлёт нотификацию юзеру.

В нём ~40 нод. Большие code ноды `Build Plan` и куча SSH-нод с ffmpeg-командами/

### 4. Реализуй одно изменение из ffmpeg

Это уже не рефакторинг, а конкретная фича. Выбери **одну** на свой вкус:

#### Вариант А: Fade-in 0.5s в начало voice-трека

Сейчас в voice-секции есть только fade-out 0.5s в конце (через `afade=t=out:st=...:d=0.5`). Добавь ещё fade-in 0.5s в начало - чтобы voice не врубался резко на первой секунде видео.

#### Вариант B: Watermark в правом-верхнем углу

Добавь текстовый ватермарк "Preview" в правый-верхний угол финального видео:
- Шрифт - любой моноширинный, размер ~24px.
- Цвет - белый, прозрачность 50%.
- Отступ от края - 20px.
- Применять для ВСЕХ финальных видео (без условий).

В ffmpeg это делается через `drawtext` filter.

#### Вариант С: Output bitrate cap 4 Мбит/с

Сейчас финальный mp4 не имеет ограничения по битрейту, размер плавает. Добавь cap `-b:v 4M -maxrate 4M -bufsize 8M`. Обоснуй, где именно ставить.

Сохрани исправленный воркфлоу как `combine-worker-mvp2-fixed.json` рядом.

## Подсказки

- В пайплайне на 16:9 финальное видео 1920×1080, на 9:16 - 1080×1920. `OUT_W` и `OUT_H` определяются в `Build Plan` Code-ноде.
- Avatar overlay использует chromakey для зелёного фона AI: `chromakey=0x79ff9b:0.12:0.05`.
- Voice сейчас обрабатывается с auto-fit темпом (atempo) и fade-out. Где именно - смотри в Build Plan.
- Запускать воркфлоу руками не нужно (для этого тебе понадобились бы реальные клипы и SSH). Главное - понять логику и аккуратно поправить ffmpeg-команду.
