## 01-mini-register.json

### Найденные проблемы

#### Проблема 1: Некорректное получение входных данных
- **Severity:** high
- **Где:** нода `Validate Input`, поле `jsCode`
- **Что не так:** в "исправленной" версии данные пытаются получить напрямую из `$json.email`, однако n8n Webhook node помещает тело POST-запроса в свойство `body`.
- **Чем грозит:** переменные будут `undefined`, что вызывает ошибку "Invalid email" и блокирует регистрацию.
- **Как фиксить:** обращаться к `$json.body.email` и т.д.

#### Проблема 2: Нет проверки на существующий email
- **Severity:** high
- **Где:** нода `Check Email Exists`
- **Что не так:** результат запроса не используется
- **Чем грозит:** создание дубликатов пользователей
- **Как фиксить:** добавить IF/Code node с остановкой выполнения

#### Проблема 3: SQL-инъекция
- **Severity:** high
- **Где:** нода `Insert User`, поле `query`
- **Что не так:** используется строковая интерполяция (`{{ }}`)
- **Чем грозит:** риск выполнения произвольных SQL-запросов
- **Как фиксить:** использовать параметризованные запросы

#### Проблема 4: Хардкод API ключа
- **Severity:** high
- **Где:** нода `Send Email`
- **Что не так:** ключ прописан прямо в workflow
- **Чем грозит:** утечка и компрометация сервиса
- **Как фиксить:** использовать credentials или переменные окружения

#### Проблема 5: Синхронный bcrypt
- **Severity:** medium
- **Где:** нода `Hash Password`
- **Что не так:** используется `hashSync`
- **Чем грозит:** блокировка event loop при нагрузке
- **Как фиксить:** использовать асинхронную версию

## 02-job-poller.json

### Найденные проблемы

#### Проблема 1: Отсутствие батчинга
- **Severity:** high
- **Где:** между `Get Pending Jobs` и `Check External Status`
- **Что не так:** все задачи обрабатываются одновременно
- **Чем грозит:** перегрузка API и базы данных
- **Как фиксить:** добавить `SplitInBatches`

#### Проблема 2: Некорректная работа с item-ами
- **Severity:** high
- **Где:** нода `Map Status`
- **Что не так:** используется `$json.id`, но нода `Check External Status` (HTTP Request) заменяет входящий `$json` своим ответом, в котором нет `id`.
- **Чем грозит:** обновление неправильных записей в БД (или попытка обновления с `id = null`).
- **Как фиксить:** использовать `$('Get Pending Jobs').item.json.id` для доступа к данным предыдущих нод.

#### Проблема 3: SQL-инъекция
- **Severity:** high
- **Где:** нода `Update Job`
- **Что не так:** строковая интерполяция
- **Чем грозит:** риск повреждения данных
- **Как фиксить:** параметризованные запросы

#### Проблема 4: Нет обработки ошибок API
- **Severity:** medium
- **Где:** нода `Check External Status`
- **Что не так:** отсутствуют retry и timeout
- **Чем грозит:** падение workflow при ошибке
- **Как фиксить:** настроить retry и timeout

#### Проблема 5: Частый cron без ограничений
- **Severity:** medium
- **Где:** `Schedule Trigger`
- **Что не так:** запуск каждую минуту без контроля нагрузки
- **Чем грозит:** перегрузка системы
- **Как фиксить:** добавить ограничения или backoff

## 03-file-uploader.json

### Найденные проблемы

#### Проблема 1: Неправильная работа с binary данными
- **Severity:** high
- **Где:** нода `Check Size`
- **Что не так:** файл читается из `json`, а не из `$binary`
- **Чем грозит:** файл не загружается или загружается пустым
- **Как фиксить:** использовать `$binary.file`

#### Проблема 2: Проверка размера через header
- **Severity:** high
- **Где:** нода `Check Size`
- **Что не так:** используется `content-length`
- **Чем грозит:** ограничение можно обойти
- **Как фиксить:** использовать `fileSize` из binary

#### Проблема 3: continueOnFail
- **Severity:** high
- **Где:** `Upload to S3`, `Save File Record`
- **Что не так:** ошибки игнорируются
- **Чем грозит:** рассинхронизация данных (файл не загружен, запись есть)
- **Как фиксить:** убрать флаг и добавить обработку ошибок

#### Проблема 4: Неочищенное имя файла
- **Severity:** high
- **Где:** `Upload to S3`
- **Что не так:** имя файла берётся от пользователя
- **Чем грозит:** path traversal и перезапись файлов
- **Как фиксить:** генерировать безопасное имя (UUID)

#### Проблема 5: SQL-инъекция
- **Severity:** medium
- **Где:** `Save File Record`
- **Что не так:** строковая интерполяция и использование `$json` после S3 ноды (данные теряются)
- **Чем грозит:** риск повреждения данных и некорректная запись (NULL в полях)
- **Как фиксить:** использовать параметризованные запросы и ссылку на ноду `Check Size` через `.item.json`

## Общее впечатление от воркфлоу

Воркфлоу покрывают базовые сценарии (регистрация, polling, загрузка файлов), однако в текущем виде не готовы к production. Основные проблемы связаны с безопасностью, отсутствием обработки ошибок и некорректной работой с данными.

Если бы проектировал с нуля, разделил бы логику на изолированные части, добавил бы централизованную обработку ошибок, обеспечил бы идемпотентность операций и контроль нагрузки.
